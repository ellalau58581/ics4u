# 🟩 HTML Semantics

## 🚀 Features

- Structuring a semantic HTML5 page using `header`, `main`, `section` and `footer`
- Creating navigation structure using `<nav>` and lists
- Organizing content using `<section>`, `<article>` and `<aside>`
- Using `<figure>` and `<figcaption>` for media content
- Implementing interactive elements using `<details>` and `<summary>`
- Understanding embedded content elements like `<iframe>` and `<object>`
- Demonstrating how semantic elements improve structure and readability

## 🧠 Key Concepts

### Semantic HTML Structure

Semantic HTML uses elements that describe their meaning rather than acting as generic containers.

Main layout structure:

- `<header>` → top of page (intro content, branding, navigation)
- `<nav>` → navigation links
- `<main>` → primary content of the page
- `<footer>` → bottom of page content

### Navigation

The `<nav>` element is used for grouping navigation links:

- Usually contains lists of links
- Helps define site structure for users and browsers

### Content Grouping Elements

Semantic grouping improves readability and organization:

- `<section>` → groups related content by topic
- `<article>` → independent content block (can stand alone)
- `<aside>` → side content related to main content

### Media Elements

Used for images, diagrams, or visual content:

- `<figure>` → self-contained media block
- `<figcaption>` → caption describing the figure

### Interactive Elements

HTML includes built-in interactive components:

- `<details>` → collapsible content container
- `<summary>` → visible title for collapsible section

### Embedded Content Elements

Used for external or embedded resources:

- `<iframe>` → embed another webpage inside current page
- `<object>` → embed external resources (PDFs, media, etc.)

### Separation of Content

Semantic HTML improves:

- Readability of code
- Accessibility (screen readers understand structure)
- SEO (search engines understand page meaning)
- Maintainability of layouts
